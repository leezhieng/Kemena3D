# Kemena3D Project Architecture

## Overview

Kemena3D is an open-source, cross-platform C++17 3D game engine with three components:

---

## 1. `kemena3d/` ŌĆö Kemena3D SDK (Core Engine Library)

The core 3D rendering engine, built as a static (or shared) C++ library.

**Tech Stack:**

- C++17, CMake (MSVC 2022, MinGW/GCC, Clang/Xcode)
- OpenGL 3.3 (optional 4.5) via GLEW
- SDL3 (windowing/input)
- ImGui + ImGuizmo (UI, docking branch)
- Jolt Physics, Recast/Detour (navigation)
- Assimp (mesh import, optional) or tinygltf (glTF/GLB only)
- AngelScript (scripting), GLM (math), miniaudio (audio)
- nlohmann/json, portable-file-dialogs, stb_image

**Key Subsystems** (from `Source/include/`):

### Asset Packaging System

The SDK includes a built-in asset packaging system for game distribution.

#### Virtual File System (`kfilesystem.h`)

`kFileSystem` is a global singleton that transparently routes file I/O to either:
- **Editor/Development mode:** Direct OS filesystem access (no change in behavior)
- **Packaged/Runtime mode:** Reads from a `.kpak` archive mounted next to the executable

```cpp
// Initialize before any asset loading
kFileSystem::init(exeDirectory, "data");

// Transparent file access ŌĆö works in both modes
if (kFileSystem::fileExists("scene.world")) { ... }
auto data = kFileSystem::readFile("Library/ImportedAssets/abc.glb");
kString json = kFileSystem::readFileString("game.config");

// For APIs that need a real OS path (e.g. Assimp mesh loading)
kString osPath = kFileSystem::resolveOSPath("Library/ImportedAssets/abc.glb");

// Check if running packaged
bool packaged = kFileSystem::isPackaged();

// Shutdown
kFileSystem::shutdown();
```

**Startup detection order:**
1. Look for `<exeName>.kpak` next to the executable
2. Look for `data.kpak` next to the executable
3. Look for `data/` folder next to the executable
4. Fall back to the executable directory itself

#### Package Format (`.kpak`)

```
[Header - 16 bytes]
  Magic:       "KPAK"         (4 bytes)
  Version:     uint32_t = 1   (4 bytes)
  Flags:       uint32_t       (4 bytes)  [bit 0 = has-compressed-entries]
  EntryCount:  uint32_t       (4 bytes)

[File Index - repeated EntryCount times]
  PathLength:        uint16_t  (2 bytes)
  Path:              char[]    (PathLength bytes, UTF-8)
  DataOffset:        uint64_t  (8 bytes)  absolute byte offset
  CompressedSize:    uint32_t  (4 bytes)  0 = uncompressed
  UncompressedSize:  uint32_t  (4 bytes)

[File Data] ŌĆö concatenated raw or LZNT1-compressed blobs
```

Paths use forward slashes relative to package root:
- `scene.world`
- `game.config`
- `Library/ImportedAssets/abc123.glb`
- `Library/Scripts/abc123.kbc`

**Compression:** On Windows, uses the built-in `RtlCompressBuffer`/`RtlDecompressBuffer` (LZNT1) from `ntdll.dll` ŌĆö zero external dependencies. Per-file compression enables O(1) random access to any asset without decompressing the entire archive.

## 2. `kemena3d-studio/` ŌĆö Kemena3D Studio (Editor)

Full-featured game editor with dockable ImGui panels.

**Architecture:**

- Single executable (`main.cpp`, ~810 lines) linking against the SDK
- ImGui docking for panel-based UI layout
- ImGuizmo for 3D manipulation gizmos (translate/rotate/scale)

**Editor Panels** (in `src/panel_*.cpp`):

| Panel               | Purpose                                                              |
| ------------------- | -------------------------------------------------------------------- |
| `PanelWorld`        | 3D viewport (orbit/pan/zoom, object picking, outlines, debug shapes) |
| `PanelInspector`    | Property inspector for selected objects/components                   |
| `PanelHierarchy`    | Scene tree view                                                      |
| `PanelProject`      | Asset browser (file management, drag-drop, thumbnail generation)     |
| `PanelConsole`      | Log/output console                                                   |
| `PanelShaderEditor` | Visual GLSL shader graph editor                                      |
| `PanelScriptEditor` | Visual node-graph script editor (AngelScript)                        |
| `PanelGame`         | Play/Pause/Stop controls, game view                                  |
| `PanelPrefab`       | Prefab editing mode                                                  |
| `MainMenu`          | File/Edit/View/Run/Help + keyboard shortcuts                         |
| `SplashScreen`      | Project creation/open dialog                                         |
| `crashhandler.cpp`  | Crash reporter (`kemena3d_crash.log` with DbgHelp stack traces)      |

**Key Features:**

- Undo/Redo system (`commands.cpp`)
- Drag-drop asset import from OS
- Object picking via color-ID rendering pass
- Selection outlines (orange) + drag-hover outlines (yellow)
- Editor camera: Alt+drag orbit, middle-mouse pan, wheel zoom
- F key: frame-focus selected with smoothstep tween
- W/E/R keys: translate/rotate/scale gizmos
- Ctrl+S/D/Delete/Z/Y shortcuts
- Octree debug visualization, NavMesh wireframe debug
- Script hot-reload while stopped
- Editor grid on "Editor Scene"
- Layout persistence (ImGui .ini serialization)
- Prefab editor with isolated offscreen rendering
- **Game Export:** Packages all assets into a single `.kpak` file for distribution

## 3. `Kemena3D-Runtime/` ŌĆö Kemena3D Runtime (Player/Export Template)

Generic, game-agnostic standalone executable. Same binary ships with every exported game; only `data/` (or `<game>.kpak`) differs.

**Flow:**

1. Initialize VFS: auto-detect `.kpak` package or `data/` folder relative to executable
2. Read `game.config` via VFS (JSON: title, width, height, fullscreen)
3. Boot engine: window ŌåÆ renderer ŌåÆ asset manager ŌåÆ world
4. Load `scene.world` via VFS
5. Start physics + scripts
6. Game loop: fixed-update scripts ŌåÆ physics step ŌåÆ update scripts ŌåÆ render
7. Cleanup + VFS shutdown on exit

**Export layout:**

```
MyGame/
Ōö£ŌöĆŌöĆ MyGame.exe              (renamed Kemena3D-Runtime)
Ōö£ŌöĆŌöĆ MyGame.kpak             (all game assets in one file)
Ōö£ŌöĆŌöĆ SDL3.dll                (runtime dependencies)
Ōö£ŌöĆŌöĆ assimp.dll
ŌööŌöĆŌöĆ ...
```

~130 lines of `main.cpp`. No ImGui/editor dependencies. Can build with `KEMENA_USE_ASSIMP=OFF` for glTF/GLB-only slim builds.

## Build System

- **SDK:** `build_sdk.py` ŌĆö interactive, builds with CMake, outputs to `Output/{Debug,Release}/`
- **Editor:** `build_editor.py` ŌĆö links against SDK output
- **Runtime:** `build_runtime.py` ŌĆö links against SDK output
- **Deps:** `download_dep.py` ŌĆö fetches all third-party dependencies
