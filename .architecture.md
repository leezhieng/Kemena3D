# Kemena3D Project Architecture

## Overview

Kemena3D is an open-source, cross-platform C++17 3D game engine with three components:

---

## 1. `kemena3d/` — Kemena3D SDK (Core Engine Library)

The core 3D rendering engine, built as a static (or shared) C++ library.

**Tech Stack:**

- C++17, CMake (MSVC 2022, MinGW/GCC, Clang/Xcode)
- OpenGL 3.3 (optional 4.5) via GLEW
- SDL3 (windowing/input)
- ImGui + ImGuizmo (UI, docking branch)
- Jolt Physics, Recast/Detour (navigation)
- Assimp (mesh import, optional) or tinygltf (glTF/GLB only)
- AngelScript (scripting), GLM (math), miniaudio (audio)
- nlohmann/json, portable-file-dialogs, stb_image

**Key Subsystems** (from `Source/include/`):

| Header                                                 | System                                                                  |
| ------------------------------------------------------ | ----------------------------------------------------------------------- |
| `kemena.h`                                             | Top-level factory API                                                   |
| `krenderer.h`                                          | Render pipeline (kDriver, framebuffers, shadows, post-process, picking) |
| `kscene.h`                                             | Scene container (objects, meshes, lights, skybox, frustum culling)      |
| `kworld.h`                                             | World management (multiple scenes, cameras, physics/script lifecycle)   |
| `kmesh.h` / `kmaterial.h` / `kshader.h` / `ktexture.h` | Asset classes                                                           |
| `klight.h`                                             | Point, spot, directional lights                                         |
| `kcamera.h`                                            | Camera types (free, FPS, third-person, orbit)                           |
| `kanimator.h` / `kanimation.h` / `kskelanimation.h`    | Animation + skeletal animation                                          |
| `kbone.h`                                              | Skeletal bone data                                                      |
| `kphysicsmanager.h` / `kphysicsobject.h`               | Jolt Physics wrapper (rigid bodies, raycasts, character controllers)    |
| `knavmanager.h` / `knavmesh.h` / `knavagent.h`         | Recast/Detour navigation (pathfinding, crowd, navmesh baking)           |
| `kscriptmanager.h` / `kscriptbindings.h`               | AngelScript integration (compile to bytecode, C++ bindings)             |
| `kscriptgraph.h`                                       | Visual node-based script graph                                          |
| `kguimanager.h`                                        | ImGui wrapper                                                           |
| `kaudiomanager.h` / `kaudio.h`                         | Audio subsystem (miniaudio)                                             |
| `kassetmanager.h`                                      | Asset loading (meshes, shaders, textures, audio)                        |
| `kprefab.h`                                            | Prefab asset support (nested, variants)                                 |
| `koctree.h`                                            | Octree spatial partitioning                                             |
| `koffscreenrenderer.h`                                 | Render-to-texture                                                       |
| `kparticle.h`                                          | Particle effects                                                        |
| `kcharactercontroller.h`                               | Character controller (Jolt)                                             |
| `kdatatype.h`                                          | Common types (vectors, matrices, UUIDs, enums)                          |
| `kexport.h`                                            | DLL export/import macros                                                |
| `kdriver.h` / `kopengldriver.h`                        | Graphics driver abstraction / OpenGL impl                               |
| `kmd5.h`                                               | MD5 hash utility                                                        |
| `kmeshgenerator.h`                                     | Procedural mesh generation                                              |

**Build Options:**

- `BUILD_SHARED_LIBS` — Static vs dynamic library
- `KEMENA_USE_ASSIMP` — Include Assimp (editor) or slim tinygltf-only (runtime)
- `KEMENA_OPENGL_45` — Target OpenGL 4.5 instead of 3.3

## 2. `kemena3d-studio/` — Kemena3D Studio (Editor)

Full-featured game editor with dockable ImGui panels.

**Architecture:**

- Single executable (`main.cpp`, ~810 lines) linking against the SDK
- ImGui docking for panel-based UI layout
- ImGuizmo for 3D manipulation gizmos (translate/rotate/scale)

**Editor Panels** (in `src/panel_*.cpp`):

| Panel               | Purpose                                                              |
| ------------------- | -------------------------------------------------------------------- |
| `PanelWorld`        | 3D viewport (orbit/pan/zoom, object picking, outlines, debug shapes) |
| `PanelInspector`    | Property inspector for selected objects/components                   |
| `PanelHierarchy`    | Scene tree view                                                      |
| `PanelProject`      | Asset browser (file management, drag-drop, thumbnail generation)     |
| `PanelConsole`      | Log/output console                                                   |
| `PanelShaderEditor` | Visual GLSL shader graph editor                                      |
| `PanelScriptEditor` | Visual node-graph script editor (AngelScript)                        |
| `PanelGame`         | Play/Pause/Stop controls, game view                                  |
| `PanelPrefab`       | Prefab editing mode                                                  |
| `MainMenu`          | File/Edit/View/Run/Help + keyboard shortcuts                         |
| `SplashScreen`      | Project creation/open dialog                                         |
| `crashhandler.cpp`  | Crash reporter (`kemena3d_crash.log` with DbgHelp stack traces)      |

**Key Features:**

- Undo/Redo system (`commands.cpp`)
- Drag-drop asset import from OS
- Object picking via color-ID rendering pass
- Selection outlines (orange) + drag-hover outlines (yellow)
- Editor camera: Alt+drag orbit, middle-mouse pan, wheel zoom
- F key: frame-focus selected with smoothstep tween
- W/E/R keys: translate/rotate/scale gizmos
- Ctrl+S/D/Delete/Z/Y shortcuts
- Octree debug visualization, NavMesh wireframe debug
- Script hot-reload while stopped
- Editor grid on "Editor Scene"
- Layout persistence (ImGui .ini serialization)
- Prefab editor with isolated offscreen rendering

## 3. `Kemena3D-Runtime/` — Kemena3D Runtime (Player/Export Template)

Generic, game-agnostic standalone executable. Same binary ships with every exported game; only `data/` differs.

**Flow:**

1. Resolve `data/` folder relative to executable
2. Read `game.config` (JSON: title, width, height, fullscreen)
3. Boot engine: window → renderer → asset manager → world
4. Load `scene.world`
5. Start physics + scripts
6. Game loop: fixed-update scripts → physics step → update scripts → render
7. Cleanup on exit

~130 lines of `main.cpp`. No ImGui/editor dependencies. Can build with `KEMENA_USE_ASSIMP=OFF` for glTF/GLB-only slim builds.

---

## Build System

- **SDK:** `build_sdk.py` — interactive, builds with CMake, outputs to `Output/{Debug,Release}/`
- **Editor:** `build_editor.py` — links against SDK output
- **Runtime:** `build_runtime.py` — links against SDK output
- **Deps:** `download_dep.py` — fetches all third-party dependencies

## Dependency Graph

```
Kemena3D-Runtime.exe ──┐
                        ├── kemena3d.lib (SDK) ──┬── OpenGL / GLEW
kemena3d-studio.exe ───┘                         ├── SDL3
                                                 ├── Assimp (or tinygltf)
                                                 ├── AngelScript
                                                 ├── Jolt Physics
                                                 ├── RecastNavigation
                                                 ├── ImGui + ImGuizmo (editor only)
                                                 ├── GLM, stb, miniaudio
                                                 └── nlohmann/json
```
